# 百度健康用户心智监测技能

## 技能概述

本技能规范了百度健康C端用户心智监测系统的完整执行流程，帮助产品团队从用户公开评论中提取心理洞察，生成可落地的产品改进建议。

**核心约束：** 所有分析基于本地文件，禁止调用任何外部搜索或抓取 API。

**质量检查：**
- [ ] 监测目标明确且可衡量
- [ ] 样本量符合预算约束
- [ ] 交付时间合理

## §2 关键词配置

**原则：**
1. **具体化** — 使用用户真实表达（"响应慢"而非"不满意"）
2. **多样化** — 覆盖正面、中性、负面三类，避免偏向性
3. **时效性** — 结合近期舆情热点

**必须包含三类词汇示例：**
```yaml
positive: ["百度健康 很靠谱", "问诊效果好", "医生专业"]
neutral:  ["百度健康怎么样", "线上问诊体验", "和医院比"]
negative: ["太贵了", "等待时间长", "不放心线上"]
```

**验证标准：** 每个类别至少2个关键词，手动命中率 > 50%

## §3 数据采集规范

> ⚠️ 本技能禁止调用任何外部API或爬虫工具，仅支持基于用户提供的本地文件分析。

**平台优先级与样本占比：**

| 优先级 | 平台 | 用户特征 | 样本占比 |
|-------|------|---------|---------|
| P0 | 小红书 | 年轻女性、健康焦虑高 | 40% |
| P1 | 知乎 | 理性决策者、深度讨论 | 30% |
| P2 | 微博 | 情绪表达强、传播力强 | 20% |
| P3 | 豆瓣 | 小众社区、长文评价 | 10% |

**时效性要求：**
- 首选：最近7天内
- 可接受：最近30天内（90%样本）
- 禁止：超过3个月的评论

**应采集 ✅ vs 应过滤 ❌：**

| ✅ 应采集 | ❌ 应过滤 |
|----------|---------|
| 含情感表达的真实评论 | 明显营销软文（带购买链接）|
| 描述具体使用场景 | 机器人评论（重复内容）|
| 涉及决策过程或障碍 | 纯事实陈述无情感表达 |

**数据格式（JSON）：**
```json
{
  "comments": [
    {
      "content": "百度健康靠谱吗？看到很多人用...",
      "platform": "小红书",
      "publish_time": "2026-04-10 14:30:00",
      "url": "https://www.xiaohongshu.com/explore/12345",
      "author": "用户1234"
    }
  ]
}
```

## §4 数据质检标准

在进行心智分析前，验证以下所有检查项：

| 检查项 | 合格标准 | 不合格处理 |
|-------|---------|-----------|
| 样本量 | ≥50条（常规） | 补充采集 |
| 时效性 | 90%在最近30天内 | 替换过期样本 |
| 平台分布 | ≥3个平台，单平台<60% | 补充弱势平台 |
| 内容质量 | 无乱码、无营销内容 | 清洗或剔除 |
| 格式规范 | 符合JSON结构 | 修正格式错误 |

## §5 心智分析框架

### 5.1 九维度提取标准

对每条评论提取以下9个维度：

#### 维度1：user_type（用户类型）
| 类型 | 判断依据 | 典型表达 |
|-----|---------|---------|
| `first_time_user` | 明确提到"第一次"、"首次" | "第一次用线上问诊" |
| `regular_user` | 描述使用经历，未提及离开 | "用了三次了" |
| `potential_user` | 处于观望期，尚未使用 | "考虑要不要用" |
| `churned_user` | 明确表示不再使用或转投竞品 | "现在改用XX了" |

#### 维度2：core_mindset（核心心智）
| 心智状态 | 情感强度范围 | 典型表达 |
|---------|------------|---------|
| `trusting` | 6-8 | "挺靠谱的"、"相信平台" |
| `hesitant` | 3-5 | "不知道靠不靠谱"、"有点担心" |
| `frustrated` | 7-10 | "气死了"、"太失望了" |
| `satisfied` | 6-9 | "很满意"、"超出预期" |
| `indifferent` | 2-4 | "无所谓"、"差不多" |
| `confused` | 3-6 | "不太懂"、"搞不清楚" |

#### 维度3：emotion_intensity（情感强度，0-10分）
- **0-2分**：几乎无情感（纯事实陈述）
- **3-4分**：轻微情感（"还行"、"一般般"）
- **5-6分**：中等情感（"挺好的"、"有点失望"）
- **7-8分**：强烈情感（"非常满意"、"很生气"）
- **9-10分**：极端情感（"爱死了"、"被气死"）

#### 维度4：primary_motivation（主要动机）
用户使用/考虑使用产品的核心驱动力，例如：
- `convenience`（便捷）、`privacy`（隐私保护）、`cost`（价格）
- `expertise`（医生专业度）、`speed`（响应速度）

#### 维度5：primary_barrier（主要障碍）
阻碍用户使用的核心因素，例如：
- `trust_issue`（信任问题）、`price`（价格贵）、`wait_time`（等待时间长）
- `interface`（操作复杂）、`efficacy_doubt`（效果存疑）

#### 维度6：decision_stage（决策阶段）
| 阶段 | 说明 |
|-----|-----|
| `awareness` | 刚了解产品，尚未形成判断 |
| `consideration` | 正在考虑是否使用 |
| `usage` | 正在使用中 |
| `evaluation` | 使用后进行评价 |
| `churn` | 已离开或考虑离开 |

#### 维度7：key_concern（核心关切）
用户最在意的1-2个关键点（自由文本，20字以内）

#### 维度8：mind_tags（心智标签）
3-5个关键词标签，概括用户心理状态

#### 维度9：filtered_out（是否过滤）
满足以下任一条件设为 `true`：
- 内容长度 < 10字
- 包含营销关键词
- 无明确情感表达
- 语义不清或乱码

### 5.2 分析质量标准

- **有效样本率 ≥ 70%**（filtered_out=false 的比例）
- **全部9个维度有数据填充**
- **情感强度分布合理**（避免全是极端值0或10）
- **人工复核触发条件：** 过滤率 > 30%，或情感强度全为极端值

## §6 洞察报告生成

### 报告结构（必需章节）

```markdown
# 百度健康用户心智监测报告
监测周期：[日期范围] | 样本量：[n条] | 有效样本：[n条]

## 执行摘要
核心发现1：[1句话]
核心发现2：[1句话]
核心发现3：[1句话]

## 用户心智全景
- 心智分布（6种心智状态占比）
- 决策旅程分布（awareness → consideration → usage → evaluation → churn）
- 情感强度分布

## 关键洞察（≥3个）
### 洞察N：[标题]
- **现象描述**：[数据支撑，引用具体数字]
- **典型案例**：[引用原始评论（脱敏）]
- **深层原因**：[用户心理分析]
- **产品建议**：[可落地的行动项]

## 高频心智标签 Top 10

## 行动建议
- 短期优化（2周内）
- 中期迭代（1个月内）
- 长期战略（季度级）
```

### 好洞察的4个标准（SADD原则）

| 标准 | 说明 |
|-----|-----|
| **S**urprising（意外性）| 揭示非显而易见的发现 |
| **A**ctionable（可落地）| 能转化为具体产品改进 |
| **D**ata-Driven（数据驱动）| 有明确数字支撑 |
| **D**eep（深度）| 超越表面，揭示"为什么" |

**示例对比：**

| ❌ 差的洞察 | ✅ 好的洞察 |
|----------|----------|
| "用户对响应速度不满" | "响应超过5分钟的问诊，用户流失率提升40%（n=23），建议优先优化高峰期医生调度" |
| "年轻用户更喜欢线上问诊" | "25-30岁女性中，因'隐私保护'选择线上问诊的占比达68%，远高于便捷性（32%），建议在营销话术中强化隐私优势" |

### 必需图表

1. 心智分布饼图（6种心智状态占比）
2. 决策阶段漏斗图
3. 情感强度分布图（按平台/用户类型分组）

## §7 质量控制三级机制

| 级别 | 执行人 | 检查内容 | 时机 |
|-----|-------|---------|-----|
| L1自检 | 分析师 | 格式规范、基础数据质量 | 每阶段完成后 |
| L2互检 | 同岗位交叉 | 分析逻辑、洞察合理性 | 报告初稿完成后 |
| L3终审 | 监测负责人 | 全流程合规性、交付质量 | 交付前 |

**常见错误预防：**

| 错误类型 | 预防措施 |
|--------|---------|
| 样本偏差（单平台>60%）| 执行平台分布检查 |
| 过度解读（<10条样本得结论）| 每个洞察样本量≥10条 |
| 心智误判 | 严格按§5.1标准执行 |
| 数据时效过期 | 强制执行30天时效规则 |

## §8 异常处理

### 常见异常快速参考

| 异常现象 | 原因 | 处理方法 |
|--------|-----|---------|
| 有效样本率 < 50% | 数据质量差/关键词不相关 | 检查过滤原因 → 优化关键词 → 补充采集 |
| 心智分布严重失衡（某类>80%）| 样本来源单一或关键词偏向 | 检查平台分布 → 补充对立关键词 → 重新分析 |
| JSON格式错误 | 手动填写格式不规范 | 使用 `jq` 验证格式 |
| 情感强度全为极端值 | 样本来源偏向极端情绪社区 | 补充知乎等理性讨论平台样本 |

## 合规性要求

- **仅分析公开内容**，不处理私密账号数据
- **隐私保护**：报告中用户真实姓名、手机号必须脱敏
- **版权尊重**：引用评论注明来源
- **禁止使用外部API或爬虫工具**
