# 9维度心智提取详细标准

> 来源：SOP-BDHC-MIND-001 § 5.1

---

## 维度1：user_type（用户类型）

| 类型 | 判断依据 | 典型表达 | 反例（不应归入） |
|------|----------|----------|----------------|
| `first_time_user` | 明确提到"第一次"、"首次"、"刚开始用" | "第一次用线上问诊，没想到…" | "我每次都用百度健康"（regular） |
| `regular_user` | 描述持续使用经历，未提及离开意图 | "用了三次了，感觉…" | "试了一下"（可能是 first_time） |
| `potential_user` | 处于观望/询问期，尚未实际使用 | "考虑要不要用，有用过的吗？" | "用过一次觉得一般"（first_time） |
| `churned_user` | 明确表示不再使用或已转投竞品 | "现在改用丁香医生了"、"卸载了" | "下次可能换个平台"（仅有意向，是 regular） |
| `unknown` | 无法从内容判断 | 纯产品功能评论，无使用频率信息 | — |

## 维度2：core_mindset（核心心智）

| 心智状态 | 情感强度范围 | 判断要点 | 典型表达 |
|----------|------------|---------|---------|
| `trusting` | 6-8 | 对平台/医生/信息表达信任、认可 | "挺靠谱的"、"相信平台"、"医生很专业" |
| `hesitant` | 3-5 | 犹豫、观望、表达不确定性 | "不知道靠不靠谱"、"有点担心"、"还在考虑" |
| `frustrated` | 7-10 | 明确表达失望、愤怒、受骗感 | "气死了"、"太失望了"、"浪费钱"、"再也不用" |
| `satisfied` | 6-9 | 表达满意、超预期、愿意推荐 | "很满意"、"超出预期"、"推荐给朋友" |
| `indifferent` | 2-4 | 无明显情感倾向，中性评价 | "无所谓"、"差不多"、"就那样" |
| `confused` | 3-6 | 表达困惑、不理解产品/流程 | "不太懂"、"搞不清楚怎么用"、"看不懂报告" |
| `unknown` | — | 无法从内容判断主要心智状态 | — |

**冲突处理**：若评论同时出现 `frustrated` 和 `satisfied`，取情感强度更高的状态。

## 维度3：emotion_intensity（情感强度，0-10）

| 分值 | 描述 | 关键词参考 |
|------|------|-----------|
| 0-2 | 几乎无情感（纯事实陈述） | "百度健康支持XX功能" |
| 3-4 | 轻微情感 | "还行"、"一般般"、"还可以"、"差不多" |
| 5-6 | 中等情感 | "挺好的"、"有点失望"、"比较满意"、"感觉还行" |
| 7-8 | 强烈情感 | "非常满意"、"很生气"、"超级好用"、"相当失望" |
| 9-10 | 极端情感 | "爱死了"、"被气死"、"强烈推荐"、"绝对不用" |

## 维度4：primary_motivation（主要动机）

用户使用或考虑使用百度健康的核心驱动力：

| 值 | 说明 | 典型表达 |
|----|------|---------|
| `convenience` | 便捷，随时可用 | "不用跑医院"、"24小时都能问" |
| `cost_saving` | 价格优势 | "比挂号便宜"、"有优惠券" |
| `privacy` | 隐私保护 | "不想被熟人看到"、"线上更私密" |
| `speed` | 快速获得回答 | "等不了线下预约"、"急着知道结果" |
| `expertise` | 专业医生资源 | "三甲医院的医生"、"权威专家" |
| `information` | 获取健康信息/科普 | "查查症状是什么"、"了解一下" |
| `unknown` | 无法判断 | — |

---

## 维度5：primary_barrier（主要障碍）

阻碍用户使用或导致流失的核心因素：

| 值 | 说明 | 典型表达 |
|----|------|---------|
| `trust_concern` | 信任问题（医生资质/平台可靠性） | "不知道是不是真医生"、"感觉不靠谱" |
| `price_concern` | 价格障碍 | "太贵了"、"不值这个价" |
| `quality_concern` | 医疗质量担忧 | "回答太敷衍"、"问了跟没问一样" |
| `privacy_concern` | 隐私担忧 | "数据会不会泄露"、"收到骚扰电话" |
| `ux_friction` | 交互体验差 | "找不到功能"、"界面太乱"、"加载慢" |
| `response_time` | 响应时间过长 | "等了很久没人回"、"医生不在线" |
| `competitor_pull` | 被竞品吸引 | "丁香医生更方便"、"好大夫有这个功能" |
| `none` | 无明显障碍（满意用户） | — |
| `unknown` | 无法判断 | — |

## 维度6：decision_stage（决策阶段）

用户在产品使用旅程中的当前阶段：

```
awareness → consideration → usage → evaluation → churn
  认知        考虑中         使用中    评估/复盘    流失/离开
```

| 阶段 | 判断依据 | 典型表达 |
|------|---------|---------|
| `awareness` | 刚了解到产品，未开始使用 | "听说百度健康…"、"有人推荐了" |
| `consideration` | 主动研究，对比中，尚未下决定 | "在考虑要不要用"、"和丁香医生比哪个好？" |
| `usage` | 正在使用中 | "刚用了一次"、"用了好几次" |
| `evaluation` | 使用后进行评价/复盘 | "用了之后感觉…"、"总结一下体验" |
| `churn` | 已停止使用或转投竞品 | "卸载了"、"现在不用了"、"改用XX了" |

## 维度7：trust_level（信任度，1-10）

| 分值 | 描述 |
|------|------|
| 1-3 | 不信任，明确表达怀疑或受骗感 |
| 4-5 | 半信半疑，持保留态度 |
| 6-7 | 基本信任，有轻微疑虑 |
| 8-10 | 高度信任，主动推荐 |

## 维度8：key_tags（心智标签，数组）

从评论中提取 2-5 个最具代表性的心智关键词，使用用户原始表达（不翻译为专业术语）。

**规范**：
- 使用名词短语（"响应速度慢"而非"慢"）
- 避免重复（不同措辞的同一问题只保留一个）
- 最多 5 个标签

**示例**：`["医生资质不透明", "报告看不懂", "价格合理", "响应及时"]`

---

## 维度9：filtered_out（是否过滤）

布尔值。满足以下任一条件时设为 `true`：
- 内容长度 < 10 字
- 包含营销关键词
- 无明确情感表达（纯事实陈述）
- 语义不清或乱码
- 重复内容（与其他条目完全相同）
