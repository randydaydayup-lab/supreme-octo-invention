## 1. user_type 判断树

```
是否明确提到"第一次"、"首次"、"刚开始"?
├── 是 → first_time_user
└── 否
    ├── 是否提到已转用其他产品或不再使用?
    │   ├── 是 → churned_user
    │   └── 否
    │       ├── 是否描述了使用经历（过去式）?
    │       │   ├── 是 → regular_user
    │       │   └── 否
    │       │       └── 处于观望/询问状态 → potential_user
```

**边界案例处理：**
- "上次用过一次" → `first_time_user`（强调首次体验）
- "以前用过，现在不用了" → `churned_user`
- "听说百度健康不错" → `potential_user`（尚未使用）

## 2. core_mindset 评分矩阵

| 文本信号 | core_mindset | 情感强度参考 |
|--------|-------------|-----------|
| 推荐给他人、重复购买意向 | `satisfied` | 7-9 |
| 描述具体好的体验 | `satisfied` | 6-8 |
| 表达信任、依赖 | `trusting` | 6-8 |
| 犹豫不决、反复比较 | `hesitant` | 3-5 |
| 提问但尚未行动 | `hesitant` | 3-5 |
| 抱怨、指责、愤怒 | `frustrated` | 7-10 |
| 失望、遗憾语气 | `frustrated` | 5-7 |
| 无明确倾向、中立陈述 | `indifferent` | 2-4 |
| 不理解功能、不知如何操作 | `confused` | 3-6 |

**注意：** 同一条评论中若有正负混合情感，取主导情感。

## 3. emotion_intensity 校准案例

| 分值 | 示例评论 | 关键信号词 |
|-----|---------|---------|
| 1-2 | "百度健康有在线问诊功能" | 纯事实陈述 |
| 3-4 | "还可以，一般般" | 还行、一般、凑合 |
| 5-6 | "挺方便的，比较满意" / "有点失望" | 挺、比较、有点 |
| 7-8 | "非常好用！强烈推荐！" / "很生气，太慢了" | 非常、强烈、很、太 |
| 9-10 | "爱死了这个功能！！！" / "被气到炸了，绝对不再用" | 爱死、气炸、绝对 + 感叹号 |

## 4. primary_motivation 分类体系

| 动机类型 | 关键词信号 | 典型表达 |
|--------|---------|---------|
| `convenience` | 方便、省时、不用出门 | "不用挂号排队" |
| `privacy` | 隐私、不想被认识、匿名 | "不想被同事看到" |

| `expertise` | 专业、权威、三甲、主任 | "医生很专业" |
| `speed` | 快、及时、立刻、马上 | "5分钟就回复了" |
| `accessibility` | 偏远、小城市、当地没有 | "我这边没有好医院" |
| `anxiety_relief` | 放心、安心、不担心了 | "心里踏实多了" |

## 5. primary_barrier 分类体系

| 障碍类型 | 关键词信号 | 典型表达 |
|--------|---------|---------|
| `trust_issue` | 靠谱吗、可信吗、真假、骗人 | "不知道医生是不是真的" |
| `price` | 贵、不划算、收费高 | "问诊费用太贵了" |
| `wait_time` | 慢、等很久、响应慢 | "等了半小时没人接" |

## 6. decision_stage 判断标准

| 阶段 | 判断依据 | 典型句式 |
|-----|---------|---------|
| `awareness` | 刚听说、第一次了解 | "刚知道有这个平台" |
| `consideration` | 正在比较、犹豫是否使用 | "在考虑要不要用" / "和XX比较了一下" |
| `usage` | 当前正在使用，描述体验 | "用起来感觉...""刚刚问诊完" |
| `evaluation` | 使用后总结评价 | "用了一段时间后..."  "总体来说..." |
| `churn` | 已经离开或明确表示要离开 | "不用了，改用XX" / "删了这个APP" |

## 7. key_concern 提炼规则

- **长度限制：** 10-20字
- **格式：** 名词短语或疑问句
- **内容：** 提炼用户最核心的一个关切点

**示例：**
- 原始评论："不知道这上面的医生是不是真的，感觉有点不靠谱"
- key_concern：`医生资质真实性存疑`

- 原始评论："等了快一个小时了还没回复，急死了"
- key_concern：`高峰期响应速度极慢`

## 8. mind_tags 生成规则

- **数量：** 3-5个标签
- **格式：** 2-4字的名词短语
- **要求：** 不重复、从不同角度描述用户心理

**常用标签词库（参考）：**
```
正面类：信任感强、体验满意、超出预期、高性价比、愿意复购
负面类：响应迟缓、价格敏感、信任危机、功能混乱、流失风险
中性类：观望中、价格比较、功能探索、首次尝试、理性评估
动机类：隐私保护、便捷需求、成本驱动、焦虑缓解、专业信任
```

## 9. filtered_out 判断规则

**设为 `true` 的条件（满足任意一条）：**

1. **内容过短：** 实际文字内容 < 10字（不含标点符号）
2. **营销内容：** 包含以下关键词之一
   - "扫码领券"、"限时优惠"、"点击链接"、"私信我"
   - "代购"、"推广"、"返利"
3. **无情感表达：** 内容为纯事实陈述，无任何情感倾向
   - 示例：`filtered_out=true` → "百度健康APP在应用商店可以下载"
4. **语义不清：** 乱码、纯表情符号、无意义字符重复

**设为 `false` 的条件：**
- 包含明确情感表达
- 描述了个人使用体验
- 内容 ≥ 10字且语义清晰
